Setting row and cell properties declaratively via grid state

We can also set row and cell proertiess through grid state, as shown for column properties on the previous page. In these cases, you also need to specify the subgrid (usually data), as row numbers and cell coordinates are local to subgrids.

state.rows and state.cells (like state.columns, discussed on the previous page) are a dynamic properties (or pseudo-properties). They are not actually a grid properties; there is are such things as grid.properties.rows and grid.properties.cells. Dynamic properties are implemented as accessors. In these cases, state.rows and state.cells have write-only implementations (just setters) that set properties on individual rowss and cells rather than in grid.properties.
To set properties on a data row using a state object, for example on the fourth data row (row index 3): 
{
    rows: {
        data: {
            3: {
                propName: 'value'
            }
        }
    }
}
To set properties on a data cell using a state object, for example the cell at the intersection of the fourth data row and the prevclose column: 
{
    cells: {
        data: {
            3: {
                prevclose: {
                    propName: 'value'
                }
            }
        }
    }
}
It is worth repeating that both of the above constructs are only useful in static grids, when you know the row numbers in advance. Otherwise, consider programmatic API calls or row metadata to set row and cell properties.
