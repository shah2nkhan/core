Setting column properties declaratively via grid state

The state object can also be used to set column, row, and cell properties declaratively, as shown below.
Setting such properties via grid.setState is only appropriate for static columns, rows, or cells. When you donâ€™t know in advance about the columns, you can set column properties programmatically at run-time. Row and cell properties can also be set programmatically; they can also be set declaratively in row metadata.

In addition to properties on the grid proper, properties on individual columns can also be set declaratively in a state object, keyed by column name.

To set properties on a column using a state object, for example the prevclose column: 
{
    columns: {
        prevclose: {
            propName: 'value'
        }
    }
}
You can see this construct being used in the State tab for the halign (horizontal alignment) property, to right-justify the numbers in the column. We will try this in Activity 5, a few pages hence.
state.columns (like state.rows and state.cells, discussed below) is a dynamic property (or pseudo-property). It is not actually a grid property; there is no such thing as grid.properties.columns. Dynamic properties are implemented as accessors. In this case, state.columns is a write-only implementation (just a setter) that sets properties in individual column's properties objects rather than in grid.properties.
Setting column properties can also be accomplished programmatically with API calls.
